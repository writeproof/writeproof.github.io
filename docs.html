Documentation

Everything you need to know about using WriteProof.

Getting Started

WriteProof is a browser-based word processor that creates cryptographic proof of your writing process. No accounts, no setup — just open the editor and start typing.

  1. Go to the Editor
  2. Start writing. Every keystroke is automatically recorded.
  3. Your document auto-saves to your browser every 5 seconds.
  4. When finished, export your document as a .json file or share a link.

Understanding the Interface

Top Bar

  • Document Title — Click to edit. Used as the filename when exporting.
  • Word Count — Updates in real-time as you type.
  • Character Count — Total characters including spaces.
  • Session Time — How long you've been writing in this session.
  • Save Status — Shows whether changes have been saved.

Bottom Bar

  • My Documents — View and switch between saved documents.
  • New — Create a new blank document.
  • Export — Download the document as a .json file with all keystroke data.
  • Import — Load a previously exported .json file.
  • Share Link — Generate a URL containing your entire document (copies to clipboard).
  • Replay — Open the document in the verification/replay view.
  • Score — View the writing authenticity score.

Keyboard Shortcuts

  • Ctrl+S / Cmd+S — Save document
  • Space (on replay page) — Play/pause replay

Exporting and Sharing

JSON Export

Click Export to download a .writeproof.json file. This file contains:

  • Your final document text
  • Every keystroke event with timestamps
  • The cryptographic hash chain
  • Document metadata (word count, duration, etc.)

Share this file with anyone who needs to verify your work.

URL Sharing

Click Share Link to generate a URL that contains your compressed document data. Anyone who opens the link can immediately replay and verify your writing.

Note: Very large documents may exceed URL length limits. In that case, use the JSON export instead.

Verifying Documents

To verify a WriteProof document:

  1. Go to the Verify page
  2. Import a .json file (drag-and-drop or click to browse)
  3. Click Play to watch the writing replay
  4. Click Verify Hashes to check the cryptographic integrity
  5. Click Score to see the authenticity analysis

If hash verification passes, the document hasn't been tampered with after export. If it fails, someone may have modified the keystroke data.

Reading the Authenticity Score

The authenticity score (0–100) analyzes writing behavior to estimate how likely the text was written by a human in real-time. It uses four metrics:

Non-linearity (0–30 points)

Measures how much the writer moved the cursor around. Human writers frequently jump back to edit earlier text. Linear, sequential typing scores lower.

Revision Intensity (0–25 points)

Measures the ratio of deletions to insertions. Human writers regularly delete and rewrite. Very few deletions is unusual.

Pause Variability (0–25 points)

Measures variation in typing rhythm. Humans pause to think, type in bursts, and have variable speeds. Uniform timing is suspicious.

Paste Analysis (0–20 points)

Penalizes large paste operations relative to total content. Small pastes (quotes, citations) are normal. Pasting an entire essay scores very low.

Score Interpretation

  • 75–100: High confidence — Strong indicators of authentic human writing
  • 50–74: Moderate confidence — Mostly consistent with human patterns
  • 25–49: Low confidence — Some concerning patterns detected
  • 0–24: Very low confidence — Patterns inconsistent with typical human writing

The score is one data point, not a definitive judgment. Use it alongside the replay and your own assessment.

Privacy & Security

WriteProof runs entirely in your browser. No data is sent to any server. Your documents are stored in your browser's localStorage and only leave your device when you explicitly export or share them. See the Privacy page for full details.

Troubleshooting

My document disappeared

Documents are stored in your browser's localStorage. Clearing browser data, using incognito/private mode, or switching browsers will remove your documents. Always export important documents as .json files.

The share link is too long

Very large documents may exceed URL length limits. Use the JSON export for documents with many keystrokes (>10,000).

Hash verification failed

This means the keystroke data has been modified after the hashes were generated. The document may have been tampered with.

The editor feels slow

Documents with very large keystroke logs (>50,000 events) may slow down. Consider starting a new document for long writing sessions.

For Instructors

How to Use WriteProof in Your Course

  1. Announce the tool: Let students know they can submit WriteProof documents for extra credibility.
  2. Request submissions: For high-stakes assignments, require WriteProof exports alongside final drafts.
  3. Review the replay: Import the student's file and watch their writing process.
  4. Check the authenticity score: Use the score as one data point (not the only factor) in evaluating work.

Sample Syllabus Language

"For major writing assignments, you may optionally submit a WriteProof verification file alongside your final draft. This demonstrates your writing process and can help build trust in your work. Visit writeproof.github.io to get started."

Sample Assignment Instructions

"Submit your final essay as a .docx file AND a WriteProof .json export. The WriteProof file should show your complete writing session. If you write across multiple sessions, export each session separately."

Technical Details

  • Hashing: SHA-256 via the Web Crypto API
  • Hash Checkpoints: Created every 10 keystrokes
  • Hash Chain: Each checkpoint includes a cumulative hash of all previous checkpoints
  • Timestamps: performance.now() for microsecond precision
  • Storage: localStorage under the key writeproof_docs
  • URL Compression: LZ-String for compact shareable URLs
  • Browser Support: Chrome, Firefox, Safari (14+), Edge — any browser with Web Crypto API
